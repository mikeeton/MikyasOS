# Dashboard Mission Control Standard

Version: 1.0  
Status: Internal Dashboard UX Standard

## Philosophy

The dashboard is not a homepage. It is Mission Control.

Within five seconds, users should understand:

- what happened
- what is happening
- what requires attention
- what should happen next

The dashboard should proactively answer questions before users dig through modules.

## Required Dashboard Capabilities

Widgets should move toward being configurable, collapsible, personalised, and eventually
draggable/resizable. Useful widgets include business health, revenue, cash flow, AI
briefing, today's priorities, recent activity, upcoming meetings, notifications, recent
documents, project progress, sales pipeline, automation status, AI recommendations, and
performance metrics.

## AI Dashboard

AI should behave like an executive assistant. It should surface grounded observations and
recommendations with explanation:

- revenue changes
- customers needing follow-up
- projects at risk
- invoices nearing overdue status
- overloaded teammates
- failed automations
- recommended actions

AI recommendations should always have clear source context and a safe action path.

## Workflow Rules

- Prefer one-click contextual actions.
- Use smart defaults from organisation, current user, previous preference, and current
  page context.
- Reveal advanced configuration progressively.
- Give immediate confirmation and undo where actions are reversible.
- Notifications should work as action centres, not just alerts.

## Personalisation

The dashboard should remember user choices such as collapsed widgets, pinned modules,
recent work, saved searches, favourite reports, theme, density, and notification
preferences.

## Review Checklist

Before approving dashboard work, ask:

1. Can users tell what needs attention in five seconds?
2. Is the next action obvious?
3. Are the widgets useful without configuration?
4. Can users hide sections they do not need?
5. Does AI explain rather than merely decorate?
6. Does the experience work with keyboard and mobile?
